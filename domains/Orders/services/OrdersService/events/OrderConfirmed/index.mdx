## Overview

The OrderConfirmed event is triggered when an order has been successfully confirmed. This event notifies downstream services that the order is ready for fulfillment. The payload has been simplified to only include the confirmed identifiers, essential line-item data, and the epoch confirmation timestamp so consumers can depend on a consistent minimal shape.
