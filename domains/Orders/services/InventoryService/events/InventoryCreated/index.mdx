## Overview
The `InventoryCreated` event is emitted when sufficient stock for a SKU becomes available and is registered in inventory. It is published to the `inventory.{env}.events` channel, giving downstream services — such as Orders — a reliable signal to re-check backordered items tied to that SKU.

### Payload example
```json title="InventoryCreated payload"
{
  "eventId": "inv-created-123",
  "productId": "PROD-456",
  "availableQuantity": 100,
  "locationId": "WH-1",
  "timestamp": "2026-06-25T09:12:00Z",
  "source": "inventory-api"
}
```

### Producing the event
Emit this event whenever a new product batch is registered or a previously out-of-stock SKU returns to availability. Include contextual metadata (location, timestamp, source) so consumers can trace why the stock appeared.

### Consuming the event
Consumers interested in stock availability (such as the Orders Service) should filter for `productId` and confirm any waiting orders for that SKU. Once inventory is confirmed, those orders can transition from `awaiting-stock` to `confirmed` and emit the corresponding order events.
